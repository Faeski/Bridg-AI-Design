03
Code-Based Design System

One source of truth, from Figma to production

The biggest friction between design and development is not tools or talent. It is inconsistency: components that look one way in Figma and another in the browser. Spacing that drifts. Colours that diverge. States that exist in code but not in the design file, or the other way around.

A code-based design system solves this by making code the single source of truth. Design tokens (colours, spacing, typography, shadows) are defined once and consumed everywhere: in Figma, in the component library, and in production. Changes propagate automatically.

This is where AI becomes transformative. Claude Code can generate, maintain, and enforce design system components. It can scaffold new components from token definitions, audit existing pages for inconsistencies, and keep design and code in lockstep without manual synchronisation.

In the coaching programme, we will set up (or evolve) a token-driven design system for Nimbl, establish workflows for AI-assisted component creation, and build habits that keep design and development aligned by default, not by effort.

Your Own Design OS

A system that makes discovery faster and prototyping smarter

The Design OS is a highly detailed, structured workflow I built for Claude Code. It is a collection of custom skills that guide AI through the entire design process: from discovery and research, through persona creation and heuristic evaluation, to competitive analysis and UX writing. Each skill is a carefully crafted instruction set that turns Claude Code into a specialist for that specific design task.

This is not a prompt template. It is a complete operating system for design work. When your team runs a discovery phase, the Design OS skills structure the research, synthesise findings, and produce actionable outputs in a fraction of the time it would take manually. That structured output then becomes golden input for the prototyping phase: Claude Code already has the context, the constraints, and the design decisions to generate prototypes that are grounded in real research, not guesswork.

You will own this. As part of the programme, I will set up the Design OS for your team, tailor the skills to Nimbl workflows, and train you on how to use, extend, and maintain it. After the engagement ends, the system is yours: a permanent capability upgrade, not a dependency.
